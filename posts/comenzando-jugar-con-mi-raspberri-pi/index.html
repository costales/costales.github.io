Comenzando a jugar con mi Raspberri PI, puesta a punto

Oct 18, 2012
raspberry

Tras instalar raspbian, vamos a afinar el sistema:

Usuario root
El usuario root no solicita contraseña. El usuario que creé era ‘pi’ y estaba asignado a estos grupos:
# groups pi
pi : pi adm dialout cdrom sudo audio video plugdev games users input

Mis pasos fueron: Asignar clave a root, crear usuario nuevo y borrar pi.
sudo su
passwd          < Meter nueva clave de root, la anterior era la del usuario 'pi'
adduser mi_usuario 
userdel -r pi

Ahora para ejecutar un comando como root:
su root         < Introducir la clave que cambiamos en el paso 2.

Asignar más memoria RAM y menos de vídeo
Por defecto asigna mucha memoria a la tarjeta de vídeo. Si sólo queremos el Raspberri como servidor, vamos a asignarle el máximo a la RAM:
sudo cp /boot/arm224_start.elf /boot/start.elf

Instalar Transmission daemon
Lo controlaremos por web ;)

sudo apt-get install transmission-daemon ; sudo /etc/init.d/transmission-daemon stop ; sudo nano /etc/transmission-daemon/settings.json

Cambiamos estos valores:

“rpc-whitelist”: “127.0.0.1? > “rpc-whitelist”: “*.*.*.*",
“rpc-password”: “password” > “rpc-password”: “tu clave”,
“rpc-username”: “username” > “rpc-username”: “tu usuario”,

Ahora:

sudo chmod g+rw /home/usuario/Downloads ; sudo chgrp -R debian-transmission /home/usuario/Downloads ; sudo /etc/init.d/transmission-daemon start 

Prueba si todo funciona en _https://yourraspi-ip:9091_.
